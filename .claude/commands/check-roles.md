# Справочник ролей и прав доступа ServiceDesk

Используй этот справочник при реализации авторизации, фильтрации данных и проверки прав.

## 8 ролей (UserRole enum)

```csharp
public enum UserRole
{
    Technician = 0,      // Техник
    Engineer = 1,         // Инженер
    ChiefEngineer = 2,   // Главный инженер
    Logist = 3,          // Логист (техник-оператор)
    ManagerTimewise = 4,  // Менеджер Timewise
    Client = 5,          // Клиент (физ. лицо)
    ManagerClient = 6,   // Менеджер клиента (сети)
    Moderator = 7        // Модератор (админ)
}
```

## Фильтрация данных по ролям

### При получении списка заявок:
```csharp
// Техник/Инженер — только назначенные
if (role is Technician or Engineer)
    query = query.Where(t => t.AssignedEngineerId == currentUserId);

// Клиент — только свои (созданные)
if (role is Client)
    query = query.Where(t => t.CreatedByUserId == currentUserId);

// Менеджер клиента — только по своей сети
if (role is ManagerClient)
    query = query.Where(t => t.ServicePoint.Client.Id == user.OrganizationId);

// Главный инженер, Логист, Менеджер TW, Модератор — все заявки
```

## Атрибут авторизации
```csharp
// Для всех авторизованных:
[RoleAuthorize]

// Для конкретных ролей:
[RoleAuthorize(UserRole.Moderator)]
[RoleAuthorize(UserRole.Logist, UserRole.Moderator)]
```

## ClaimsPrincipalExtensions
```csharp
User.GetUserId()   // int — ID текущего пользователя
User.GetRole()     // UserRole — роль текущего пользователя
```

## Авторизация через SMS
1. Пользователь вводит номер телефона
2. Система отправляет SMS-код (не чаще 1 раз/мин)
3. Максимум 5 попыток ввода кода
4. После 5 неудач — блокировка 20 минут
5. Сессия бессрочная (до явного выхода)
6. Модератор может принудительно завершить сессию
7. Логирование всех попыток входа

## Клиент → Менеджер клиента
Пользователь регистрируется как «Клиент» (роль по умолчанию).
Модератор привязывает его к организации → роль меняется на «Менеджер клиента».
К одной организации может быть привязано несколько менеджеров.
